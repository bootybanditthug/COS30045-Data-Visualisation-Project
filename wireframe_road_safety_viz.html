Key design decisions
Colour ramp: single sequential grey-to-dark scale — avoids misleading dual-hue diverging palette. Colourblind safe.

Year slider: D3 transition animates state fills on drag, making temporal change immediately visible.

Tooltip: appears on hover, shows state name, year, raw test count and positivity rate.

Metric toggle: allows user to switch between alcohol and drug enforcement without reloading.
Hover state → tooltip · Drag year slider → animated transition · Click metric pills → data updates
Design justification — Choropleth
Choropleth maps are a non-standard D3 chart (satisfying the Complexity rubric). The general public can immediately identify geographic patterns without needing to read a table. The year slider makes temporal change perceptible — this directly supports the storytelling criterion by letting users discover whether enforcement intensity shifted across states over time.

Design justification — Dual-axis line chart
Two y-axes are used to overlay enforcement volume (tests conducted) and outcome (positivity rate) on one chart. This directly answers the central question — if increased testing correlates with falling positivity. Annotations mark key peaks to guide a general audience who may not interpret raw trends. The state selector keeps the chart non-cluttered while allowing exploration.

Design justification — Slope chart
A slope chart (non-standard) is chosen over a grouped bar chart because it makes directional change — up or down — immediately legible for a general audience. The two user-selected years allow comparison of any period, not just start-to-end. Colour encodes direction of change using accessible, single-ramp grays — no red/green that could mislead colourblind users.

Design justification — Streamgraph
A streamgraph is chosen over a stacked bar chart to show the organic flow of fine composition over time — it communicates growth, shrinkage, and relative proportion simultaneously. This is a non-standard D3 chart using d3.stack() with a wiggle offset, directly satisfying the Complexity criterion. The general audience can immediately see which offence categories have grown in enforcement focus.

3.1b — User Storyboard (6-step interaction flow)
01
User arrives
Lands on hero. Reads headline question. Sees three key stat callouts counting up via animation. Scrolled-to-explore hint visible below.
02
Map exploration
User drags year slider — map colours animate. Hovers over NSW → tooltip shows test count and positivity rate. Switches metric from alcohol to drug tests.
03
Trend discovery
User selects their home state. Sees both lines — more tests, but positivity rate also changed. Annotation labels explain the peak year. Toggles alcohol vs drug on legend.
04
State comparison
User picks two years on the slope chart. Sees which states improved and which worsened. Hovers a state line to see exact before/after positivity rates.
05
Fines picture
User sees the streamgraph — speeding dominates but mobile phone fines grew sharply post-2012. Clicks the mobile phone band to isolate it. Tooltip confirms the trend.
06
Key takeaway
User reads conclusion cards — plain language summary of what the data shows. Links to BITRE data source for further reading.

Accessibility & Responsive Design Notes (Rubric: Iteration & Validation)
Colour accessibility All charts use a single sequential grey ramp — no red/green pairs that fail Deuteranopia or Protanopia tests. Colour is never the sole encoding — shape (line vs. dashed), position, and labels provide redundant encoding.
Typography Minimum font size 12px for body text, 10px for axis labels. High contrast text on all backgrounds (WCAG AA minimum 4.5:1 ratio). Responsive font scaling via CSS clamp().
Responsive layout CSS Grid with auto-fit columns collapses to single-column on tablet and mobile. Charts resize via D3 viewBox and SVG width="100%". Slider controls remain touch-friendly (min 44px tap target).
Keyboard navigation Nav links accessible via Tab key. Chart filters (pills) have role="button" and keyboard event listeners. Tooltips also triggered via focus events.
